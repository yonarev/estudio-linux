	lo que significa que varios usuarios pueden

	los miembros de un grupo tiene Privilegios

						S: El proceso está en estado de suspensión (sleep).
						D: El proceso se encuentra en estado de espera inactivo (disk wait).
						Z: El proceso está en estado zombie (un proceso que ha terminado pero aún no ha sido liberado por su padre).
						T: El proceso está en estado de seguimiento (traced).
						W: El proceso está en espera de un evento de E/S (waiting in I/O).
						X: El proceso está en estado de ejecución no interrumpible (un proceso que no puede ser interrumpido por señales).
						P: El proceso está en estado de espera de un hijo (parent process waiting for a child).
						+: El proceso se encuentra en estado de subproceso multihilo (multi-threaded subprocess).
						l: El proceso tiene un líder de sesión controlante (has a controlling session leader).
						<: El proceso está en estado de ejecución en segundo plano (running in background).

			Señales de Linux y sus descripciones:
				Señal	Descripción
				1 (HUP)	Significa colgar. Se envía automáticamente cuando se cierra una ventana de terminal que inició un proceso antes de finalizar el proceso.
				2 (INT)	Significa interrupción, y envía la misma señal que se usa cuando presionamos Ctrl-C en la terminal, que generalmente termina el proceso.
				9 (KILL)	Significa matar, no se envía al proceso, sino al kernel del sistema operativo, que inmediatamente detiene y finaliza el proceso.
				15 (TERM)	Significa terminar. El proceso lo recibirá y terminará solo. Es la señal predeterminada enviada por kill.
				18 (CONT)	Significa continuar. Se puede utilizar para reanudar un proceso detenido.
				19 (STOP)	Significa detener, no se envía al proceso, sino al kernel del sistema operativo, que inmediatamente detiene el proceso sin terminar.
				20 (TSTP)	Significa parada terminal. A diferencia de la señal STOP, esta señal se envía al programa, es la señal enviada por el terminal cuando se presiona CTRL-Z

				HUP (Hangup): Esta señal se envía cuando se cierra una ventana de terminal que inició un proceso. El proceso puede recibir esta señal y finalizar por su cuenta o puede ignorarla.
				INT (Interrupt): Esta señal se envía cuando se presiona Ctrl-C en la terminal. La mayoría de los procesos terminarán cuando reciban esta señal, pero algunos programas pueden ignorarla o manejarla de manera diferente.
				KILL (Kill): Esta señal se envía para matar a un proceso inmediatamente. El proceso no tendrá la oportunidad de finalizar por su cuenta y se cerrará por la fuerza.
				TERM (Terminate): Esta señal se envía para indicar a un proceso que debe terminar. El proceso recibirá esta señal y tendrá la oportunidad de finalizar de forma ordenada.
				CONT (Continue): Esta señal se utiliza para reanudar un proceso que se ha detenido. El proceso se reanudará y continuará ejecutándose.
				STOP (Stop): Esta señal se utiliza para detener un proceso. El proceso se detendrá y no podrá continuar ejecutándose hasta que se envíe la señal CONT.
				TSTP (Terminal Stop): Esta señal se utiliza para detener un proceso desde la terminal. El proceso se detendrá y no podrá continuar ejecutándose hasta que se presione Ctrl-Z de nuevo.
